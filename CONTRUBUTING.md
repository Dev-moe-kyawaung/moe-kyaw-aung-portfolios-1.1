# Contributing to Moe Kyaw Aung Portfolio 1.1

Thanks for your interest in contributing! 🎉

## How to Contribute

1. **Fork** the repository
2. **Create a branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes** — please keep code style consistent (semantic HTML5, Tailwind utility classes, commented JS)
4. **Test locally** by opening `index.html` in a browser across desktop/tablet/mobile widths
5. **Commit**: `git commit -m "feat: describe your change"`
6. **Push**: `git push origin feature/your-feature-name`
7. **Open a Pull Request** describing what you changed and why

## Commit Message Convention

We use conventional commits:
- `feat:` — new feature
- `fix:` — bug fix
- `docs:` — documentation only changes
- `style:` — formatting, no code change
- `refactor:` — code change that isn't a fix or feature
- `perf:` — performance improvement

## Code Style

- Use semantic HTML5 elements
- Keep Tailwind classes readable — group related utilities together
- Comment any new JavaScript function blocks
- Test dark/light mode and Burmese/English toggle after UI changes
